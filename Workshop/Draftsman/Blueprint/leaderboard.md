# Leaderboard Blueprint

## Purpose

Define a `/leaderboard` command that renders top-N trainers for a given timeframe and circle.

## Inputs

- `circle_id` (optional)
- `period` (daily | weekly | monthly)
- `limit` (default 10)

## Outputs

- HTML/PNG leaderboard card
- Optional Discord embed summary with top 3

## Acceptance criteria

- Handles empty circle (returns friendly message)
- Respects rate limits and caches queries for 10m

## Implementation notes

- Use uma.moe rankings endpoints when available
- Provide offline mock data for renderer tests

### Optional Enhancements

- Show the trainer's **Discord avatar** (48×48 px) next to each trainer name.
- Add a small **▲ / ▼** indicator beside each trainer to show movement since the previous period.
- Color the **Fan Gain** value:
  - 🟢 Positive growth
  - 🔴 Negative growth (if applicable)
- Highlight the requesting user's row with a subtle border or accent color, even if they are outside the top 3.

This layout scales well from Top 10 to Top 25 (with a taller canvas) and immediately emphasizes the ranking data users are looking for.
