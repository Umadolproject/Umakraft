## Purpose

This blueprint defines the visual and data structure for the `/circle_master` deliverable.

It specifies how the day-by-day Top 3 fan-gain contributors for the current month are rendered as a calendar-grid image card.

---

## Product Overview

The `/circle_master` card is a monthly calendar grid showing the Top 3 daily fan-gain contributors for each day of the current month. It is targeted at members who want to see daily recognition and track who has been leading the circle each day.

## Data Contract

```json
{
  "meta": {
    "circleId": "string",
    "circleName": "string",
    "month": "string (e.g. July 2026)",
    "viewingDay": "number",
    "generatedAt": "ISO timestamp"
  },
  "days": [
    {
      "day": "number",
      "dateLabel": "string (e.g. Jul 01)",
      "isToday": "boolean",
      "isFuture": "boolean",
      "top3": [
        {
          "rank": "number (1–3)",
          "trainerName": "string",
          "gain": "number"
        }
      ]
    }
  ]
}
```

---

## Validation Rules

A valid deliverable must satisfy:

- `meta.circleId` — present, non-empty string
- `meta.circleName` — present, non-empty string
- `meta.month` — present, non-empty string
- `meta.viewingDay` — present, integer 1–31
- `days` — present, array with at least one entry
- PNG buffer — non-null, size > 0

---

## Formatting Rules

- Gain values formatted with locale comma separators and `+` prefix
- `generatedAt` rendered as `Today HH:MM` if same day, otherwise `DD MMM HH:MM`
- Empty `top3` array for a past day renders as `—` across all columns
- Future days render all columns as `—` with muted background

## Optional Enhancements

## Pipeline Ownership

| Stage | Department | Responsibility |
|-------|-----------|---------------|
| 1 — Umamoe | Miner | Fetch daily fan gain records for all active members |
| 1 — Umamoe | Inspector | Validate daily gain record structure and types |
| 1 — Umamoe | Vault | Store validated daily gain records |
| 2 — Refinery | Refiner | Aggregate Top 3 per day, compute daily rankings |
| 2 — Refinery | Compiler | Assemble compiled circle master product |
| 2 — Refinery | Depot | Persist and serve compiled product |
| 3 — Workshop | Draftsman | Own this blueprint |
| 3 — Workshop | Fabricator | Render circle master card from compiled product |
| 3 — Workshop | Validator | Approve rendered deliverable before release |
| 4 — Distribution | Commands | Intake and validate `/circle_master` interaction |
| 4 — Distribution | Coordinator | Orchestrate pipeline, retrieve deliverable |
| 4 — Distribution | Dispatcher | Deliver card to Discord |

## Error Handling

| Condition | Response |
|-----------|----------|
| No data for requested day | `❌ No contributor data is available for that day yet.` |
| Day out of range | `❌ Day must be between 1 and 31.` |
| Day in the future | `❌ Day cannot be in the future.` |
| Circle not found | `❌ Circle not found.` |
