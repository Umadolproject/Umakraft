## Purpose

This blueprint defines the visual and data structure for the `/total_circlefan_gain` deliverable.

It specifies how the circle's total accumulated fan gain for the current month is rendered — combining all active members' contributions into a single aggregate image card.

---

## Product Overview

The `/total_circlefan_gain` card is a circle-wide summary showing the combined fan gain of all active members for the current month. It includes a large aggregate total, a ranked contribution breakdown per member, and a summary statistics block.

## Data Contract

```json
{
  "meta": {
    "circleId": "string",
    "circleName": "string",
    "period": "string (e.g. July 2026)",
    "activeMemberCount": "number",
    "generatedAt": "ISO timestamp"
  },
  "total": "number",
  "contributors": [
    {
      "rank": "number",
      "trainerName": "string",
      "monthlyGain": "number",
      "sharePercent": "number"
    }
  ]
}
```

---

## Validation Rules

A valid deliverable must satisfy:

- `meta.circleId` — present, non-empty string
- `meta.circleName` — present, non-empty string
- `meta.period` — present, non-empty string
- `total` — present, integer ≥ 0
- `contributors` — present, array (may be empty)
- PNG buffer — non-null, size > 0

---

## Formatting Rules

- All gain numbers formatted with locale comma separators
- Share percentages formatted to one decimal place, e.g. `17.2 %`
- `generatedAt` rendered as `Today HH:MM` if same day, otherwise `DD MMM HH:MM`
