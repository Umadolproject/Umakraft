# Contracts

**Interface Contracts & Specifications**

This folder defines the contracts that govern communication between pipeline departments.

## Purpose

Contracts specify:
- Input/output types
- Function signatures
- Data structures
- Error codes
- Versioning rules

## Principle

Contracts are stable. They define the boundaries between departments.

Changing a contract requires an ADR in `GOVERNANCE/ARCHITECTURE_DECISIONS.md`.
