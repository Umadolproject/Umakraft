AI Option A optimization package

Summary
- Hardened the local AI service with timeout protection, response validation, degraded mode, richer telemetry, cache-key isolation, and `/status` AI health reporting.
- Built on top of the prior slash-command refactor package.

Modified files
- `AI/Configuration.js`
- `AI/aiService.js`
- `AI/cache.js`
- `AI/documentSearch.js`
- `Distribution/Coordinator/actions/aiGateway.js`
- `Distribution/Coordinator/actions/status.js`

Implemented improvements
1. Cache correctness
   - `AI/cache.js` now keys by query + command mode + retrieval mode.
   - Added cache metrics: hits, misses, writes, evictions.

2. Hard generation timeout
   - `AI/Configuration.js` adds `AI_LOCAL_GENERATION_TIMEOUT_MS`.
   - `AI/aiService.js` wraps local model generation in a timeout boundary.

3. Local-path validation
   - `AI/aiService.js` now validates local-model output with `AI/ResponseValidator.js` before sending it to Discord.
   - Includes one regeneration attempt when validation asks for a retry.

4. Degraded mode / graceful fallback
   - Added repeated-failure tracking, configurable failure threshold, and degraded mode window.
   - When degraded or generation fails, the service can fall back to documentation-only answers instead of silently failing.

5. AI observability
   - Added request, retrieval, generation, timeout, validation-failure, and docs-only fallback counters.
   - Added retrieval and generation duration tracking.
   - Added AI runtime status export for bot health reporting.

6. `/status` AI health visibility
   - `Distribution/Coordinator/actions/status.js` now exposes AI provider, health state, cache metrics, average retrieval/generation times, timeout count, validation failures, and fallback usage.

7. Local gateway correctness improvement
   - `Distribution/Coordinator/actions/aiGateway.js` now classifies first and preserves the dedicated message-generation path before local-provider routing.

New configuration knobs
- `AI_LOCAL_GENERATION_TIMEOUT_MS` (default 45000)
- `AI_LOCAL_FAILURE_THRESHOLD` (default 3)
- `AI_LOCAL_DEGRADED_MS` (default 300000)
- Existing knobs retained: `AI_LOCAL_MODEL`, `AI_LOCAL_DEVICE`, `AI_LOCAL_IDLE_TIMEOUT_MS`, `AI_LOCAL_MAX_CONCURRENCY`

Expected outcomes
- Slow or wedged local generation no longer hangs indefinitely.
- Repeated model failures trigger a controlled degraded mode.
- Cache collisions between different command shapes are prevented.
- Local-model answers are screened before delivery.
- `/status` gives actionable AI runtime diagnostics.

Important caveat
- This package was validated at source level only. The sandbox did not provide a runnable Node runtime or live Discord/Railway execution path, so runtime deployment validation still needs to happen after upload.
