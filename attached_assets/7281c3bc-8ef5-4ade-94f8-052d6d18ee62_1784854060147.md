Pipeline Phase 3 package

Summary
- Preserved the stage roles while adding operator tooling and stronger observability around Broadcast and cross-stage orchestration.
- Added dead-letter inspection/replay support, archive metrics, and deeper shared pipeline primitives.
- Extended SQLite archive querying with better indexes for dead-letter and history analysis.

Implemented improvements
1. Dead-letter operator tooling
   - Added archive support for listing dead-letter records and replaying a dead-letter record.
   - Added slash commands:
     - `deadletter_inspect`
     - `deadletter_replay`
   - Added coordinator actions and handlers for both.

2. Broadcast observability
   - Archive now exposes `getStats()` for total/completed/incomplete/dead-letter/history counts.
   - `/status` now includes Broadcast Archive health summary.

3. SQLite query/index improvements
   - Added dead-letter and outcome/time indexes to Archive SQLite storage.
   - Improved dead-letter inspection query path.

4. Stronger pipeline runtime policy
   - Added `BROADCAST_DEADLETTER_INSPECT_LIMIT` to the shared runtime config.

5. Shared pipeline consistency
   - Phase 2 primitives remain in use and are extended by Phase 3 for operator workflows.
