## Purpose

This blueprint defines the visual and data structure for the `/link_list` deliverable.

It specifies how the paginated list of all Discord-to-Uma.moe account links is rendered as an image report for administrative auditing.

---

## Product Overview

The `/link_list` card is an admin-facing paginated table image showing every linked member — their Discord identity alongside their associated Uma.moe trainer name and ID. It is used to audit link coverage, spot missing links, and verify correct assignments.

## Data Contract

```json
{
  "meta": {
    "totalLinked": "number",
    "currentPage": "number",
    "totalPages": "number",
    "generatedAt": "ISO timestamp"
  },
  "rows": [
    {
      "index": "number",
      "discordHandle": "string",
      "trainerName": "string",
      "trainerId": "string | number"
    }
  ]
}
```

---

## Validation Rules

A valid deliverable must satisfy:

- `meta.totalLinked` — present, integer ≥ 0
- `meta.currentPage` — present, integer ≥ 1
- `meta.totalPages` — present, integer ≥ 1
- `rows` — present, array (may be empty on page 1 if no links exist)
- PNG buffer — non-null, size > 0

---

## Formatting Rules

- Rows numbered globally: row 1 on page 2 of 25-per-page = index 26
- Discord handle shown as `@handle`
- `generatedAt` rendered as `Today HH:MM` if same day, otherwise `DD MMM HH:MM`
- Trainer ID rendered in monospace
