## Purpose

This blueprint defines the visual and data structure for the `/club_gain` deliverable.

It specifies how a 30-day club gain history report is rendered when requested via Discord — covering daily progress, running totals, and performance statistics.

---

## Product Overview

The `/club_gain` report is a spreadsheet-style summary card for a club's fan gain history over the last 30 days. It is targeted at club leaders and officers who need to review daily progress and performance trends at a glance.

---

## Command

```
/club_gain
```

---

## Parameters

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `club` | String | No | Requester's club | View another club (Admin only) |
| `days` | Integer | No | 30 | Number of days to display (max: 30) |

## Data Contract

The blueprint expects:

- `meta`
  - `clubId`
  - `clubName`
  - `trainerId`
  - `generatedAt`
  - `periodDays` (default 30)
- `rows`: array of
  - `date` — formatted display date (e.g. `Jul 01`)
  - `dailyGain` — integer
  - `runningTotal` — integer
- `summary`
  - `totalGain`
  - `averageDailyGain`
  - `highestGain`
  - `lowestGain`

## Pipeline Ownership

| Stage | Department | Responsibility |
|-------|-----------|---------------|
| 1 — Umamoe | Miner | Fetch raw club gain records from uma.moe |
| 1 — Umamoe | Inspector | Validate gain record structure and field types |
| 1 — Umamoe | Vault | Store validated gain records |
| 2 — Refinery | Refiner | Compute daily deltas, running totals, summary stats |
| 2 — Refinery | Compiler | Assemble compiled club gain product |
| 2 — Refinery | Depot | Persist and serve compiled product |
| 3 — Workshop | Draftsman | Own this blueprint |
| 3 — Workshop | Fabricator | Render spreadsheet card from compiled product |
| 3 — Workshop | Validator | Approve rendered deliverable before release |
| 4 — Distribution | Commands | Intake and validate `/club_gain` interaction |
| 4 — Distribution | Coordinator | Orchestrate pipeline, retrieve deliverable |
| 4 — Distribution | Dispatcher | Deliver card to Discord |

---

## Error Handling

| Condition | Response |
|-----------|----------|
| No data found | `❌ No club gain data found for the last 30 days.` |
| Invalid club | `❌ Club not found.` |
| Permission denied | `❌ You do not have permission to use this command.` |

---

## Performance Requirements

- Response time under 2 seconds.
- Maximum 30 records returned per request.
- Queries optimized by date and club ID.
- Results cached for 10 minutes.
