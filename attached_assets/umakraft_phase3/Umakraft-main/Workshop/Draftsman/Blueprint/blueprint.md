## Purpose

This blueprint defines the image-report base product for the Workshop.

It describes the layout, components, visual structure, and presentation rules required to render a report image from pipeline data. It serves as the canonical base template for automated monitoring, review dashboards, alerts, and archive snapshots.

---

## Product Overview

The image report is a snapshot card that summarizes a pipeline result and highlights the most important metrics, trends, and flags. It is designed for general-purpose monitoring deliverables that do not have a dedicated command-specific blueprint.

---

## Layout

1. Header
   - Product title
   - Source / pipeline identifier
   - Timestamp
   - Status badge
2. Summary metrics band
   - Daily gain
   - Weekly gain
   - Monthly gain
   - Rank change
3. Main visual section
   - Trend sparkline or bar chart
   - Primary status icon
4. Flag and detail panel
   - Active flags
   - Key profile attributes
   - Notes or alert summary
5. Footer
   - Blueprint version
   - Generated by
   - Raw payload reference

## Component Definitions

### Header

| Field | Type | Description |
|-------|------|-------------|
| `title` | String | Report title |
| `subtitle` | String | Secondary description |
| `generatedAt` | ISO timestamp | Generation time |
| `status` | Enum | `OK`, `Warning`, `Alert`, `Unknown` |
| `sourceId` | String | Pipeline source identifier |

### Summary Metrics Band

| Field | Type | Description |
|-------|------|-------------|
| `dailyGain` | Number | Daily fan gain |
| `weeklyGain` | Number | Weekly fan gain |
| `monthlyGain` | Number | Monthly fan gain |
| `rankChange` | Number | Rank position change (positive = improved) |

### Main Visual Section

| Field | Type | Description |
|-------|------|-------------|
| `trendData` | Array of `{ label, value }` | Trend data points |
| `primaryChartType` | Enum | `sparkline`, `bar`, `line` |
| `icon` | Enum | `up`, `down`, `flat`, `warning` |

### Flag and Detail Panel

| Field | Type | Description |
|-------|------|-------------|
| `flags` | Array | Each: `{ code, label, severity, description }` |
| `profileDetails` | Array | Each: `{ label, value }` |
| `note` | String | Free-text alert or summary note |

### Footer

| Field | Type | Description |
|-------|------|-------------|
| `blueprintVersion` | String | Version of this blueprint |
| `rendererVersion` | String | Version of Fabricator renderer used |
| `payloadReference` | String | Reference ID to raw source payload |

## Canvas and Styling

| Property | Value |
|----------|-------|
| Canvas dimensions | 1200 × 800 px (desktop card) or scalable SVG |
| Base font | 16 px sans-serif |
| Palette | Neutral background, high-contrast metric colors, status accent colors |
| Layout grid | 12-column responsive grid |
| Outer margin | 24 px |
| Internal gutters | 16 px |

---

## Rendering Rules

- Use SVG primary output to preserve crisp text and vector shapes.
- Support export to PNG at 2× scale.
- Render text labels with high contrast.
- Avoid more than 5 colors besides the base palette.
- Highlight active flags in the detail panel.
- Place the status badge in the top-right of the header.

---

## Data Contract

The blueprint expects a JSON payload with these sections:

- `meta`
  - `id`
  - `generatedAt`
  - `source`
  - `status`
- `metrics`
  - `dailyGain`
  - `weeklyGain`
  - `monthlyGain`
  - `rankChange`
- `trend`
  - `points`: array of `{ label, value }`
- `flags`: array of `{ code, label, severity, description }`
- `profile`: array of `{ label, value }`

---

## Validation Rules

A valid deliverable must satisfy:

- Required fields present: `meta.id`, `meta.generatedAt`, `meta.status`
- Numeric metrics within reasonable bounds
- `trend.points` non-empty array
- Each flag entry includes `code`, `label`, and `severity`

## Pipeline Ownership

| Stage | Department | Responsibility |
|-------|-----------|---------------|
| 2 — Refinery | Compiler | Assemble compiled image report product |
| 2 — Refinery | Depot | Persist and serve compiled product |
| 3 — Workshop | Draftsman | Own this blueprint |
| 3 — Workshop | Fabricator | Render image report card from compiled product |
| 3 — Workshop | Validator | Approve rendered deliverable before release |
| 4 — Distribution | Coordinator | Retrieve deliverable and route to Dispatcher |
| 4 — Distribution | Dispatcher | Deliver rendered report to Discord |

---

## Error Handling

| Condition | Response |
|-----------|----------|
| Required fields missing | Validator rejects deliverable — not released |
| Trend data empty | Render card without chart section; do not fail |
| Flag severity unknown | Default to `low`; log classification warning |

## Notes

- This blueprint is the general-purpose base template. Command-specific blueprints (fan_gain, profile, etc.) take precedence for their respective deliverables.
- The Fabricator is responsible for SVG/PNG generation — the blueprint is format-specification only.
- Keep the source data payload separate from the visual layout specification.
