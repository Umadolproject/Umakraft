## Purpose

This blueprint defines the visual and data structure for the `/memberlist` list-mode deliverable.

In list mode, two image reports are generated: one for current active members and one for alumni (former members). Each is a roster table card sorted by join date.

---

## Product Overview

The `/memberlist` list-mode output is a pair of image roster tables used by circle members and administrators to review the full membership history — who is active now, and who has been a member in the past.

---

## Command

```
/memberlist list:true
```

## Data Contract

```json
{
  "meta": {
    "circleId": "string",
    "circleName": "string",
    "generatedAt": "ISO timestamp"
  },
  "activeMembers": [
    {
      "rank": "number",
      "trainerName": "string",
      "discordHandle": "string | null",
      "joinedAt": "YYYY-MM-DD"
    }
  ],
  "alumni": [
    {
      "rank": "number",
      "trainerName": "string",
      "discordHandle": "string | null",
      "joinedAt": "YYYY-MM-DD",
      "lastActiveAt": "YYYY-MM-DD"
    }
  ]
}
```

---

## Validation Rules

A valid deliverable must satisfy:

- `meta.circleId` — present, non-empty string
- `meta.circleName` — present, non-empty string
- `activeMembers` — present, array (may be empty)
- `alumni` — present, array (may be empty)
- Both PNG buffers — non-null, size > 0

---

## Formatting Rules

- Dates formatted as `YYYY-MM-DD`
- Discord handle shown as `@handle`; unlinked members show `—`
- Rows numbered sequentially starting at 1 within each image

---
