## What "Bad Request" means

A **400 Bad Request** is almost never a network issue.

It usually means:

* invalid JSON
* missing required field
* wrong endpoint
* wrong model name
* empty input
* invalid API key format
* unsupported parameter

## Common causes

### 1. Wrong model

Example:

```js
model: "text-embedding-ada-002"
```

or

```js
model: "embedding"
```

If your provider doesn't support that model you'll get 400.

---

### 2. Empty text

If you're indexing files:

```js
content = ""
```

and then

```js
embeddings.create({
    input: content
})
```

Some providers reject empty strings.

---

### 3. Wrong endpoint

For OpenAI-compatible APIs:

```
POST /v1/embeddings
```

If you're sending embeddings to

```
/chat/completions
```

you'll get 400.

---

### 4. Missing required field

For example:

```js
{
    model: "...",
}
```

without

```js
input
```

returns 400.

---

### 5. Invalid provider

If you're using OpenRouter, Together AI, Groq, Gemini, Voyage, etc., each has slightly different supported models and endpoints.

---

## The biggest problem with your logs

This line:

```text
AI Repository Engine init failed (AI commands degraded): Bad Request
```

is **throwing away the real error**.

Instead of logging only:

```
Bad Request
```

you should log:

```js
console.error(error.response?.status);
console.error(error.response?.data);
console.error(error.stack);
```

or

```js
console.error(JSON.stringify(error.response.data, null, 2));
```

If you're using `fetch`:

```js
const res = await fetch(...);

if (!res.ok) {
    console.error(await res.text());
}
```

That usually prints something like:

```json
{
  "error": {
    "message": "Model not found"
  }
}
```

or

```json
{
  "error": {
    "message": "Missing required parameter: input"
  }
}
```

which immediately identifies the issue.

## The Discord warning

This is unrelated:

```text
DeprecationWarning:
The ready event has been renamed to clientReady
```

Simply change:

```js
client.on("ready", ...)
```

to

```js
client.on("clientReady", ...)
```

This won't fix the AI issue, but it will remove the warning and keep your bot compatible with Discord.js v15.
